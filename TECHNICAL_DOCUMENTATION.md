# Technical Documentation

## 1. Project Overview
The **Shadcn Landing Page Template** is a modern, responsive, and highly customizable landing page designed to kickstart web projects. It provides a solid foundation for building landing pages with dark/light mode support out of the box.

## 2. Technology Stack
This project leverages a modern frontend technology stack:
- **Core Framework**: React (v18.3.1)
- **Language**: TypeScript for static type checking and robust code.
- **Build Tool**: Vite (v5.2.13) for fast development server and optimized production builds.
- **Styling**: Tailwind CSS (v3.4.4) for utility-first CSS styling.
- **Component Library**: Shadcn/UI, which provides accessible, unstyled components built on top of **Radix UI** primitives.
- **Icons**: `lucide-react` and custom SVG components.

## 3. Project Architecture and Directory Structure
- `src/main.tsx`: The application entry point. It wraps the app in a `ThemeProvider` to manage global dark/light modes.
- `src/App.tsx`: The primary container component that sequentially imports and renders all the landing page sections (Hero, About, Features, Pricing, etc.).
- `src/components/`: Contains all the modular sections of the landing page:
  - **Layout Sections**: `Navbar`, `Hero`, `Features`, `Pricing`, `Footer`, etc.
  - **UI Primitives**: Located in `src/components/ui/` (e.g., Accordion, Button, Dialog) which are generated by Shadcn/UI.
  - **Theme Management**: `theme-provider.tsx` and `mode-toggle.tsx` handle the dark/light mode switching.
- `src/lib/`: Contains utility functions.
  - `utils.ts`: Contains the `cn` function, which combines `clsx` and `tailwind-merge` to intelligently merge Tailwind classes without conflicts.

## 4. Functions and API Usage
Currently, the template is a **Static Frontend UI**.
- **APIs Used**: There are no external REST APIs, GraphQL endpoints, or database connections configured by default. All data (like pricing plans, team members, and testimonials) is hardcoded within the respective components in `src/components/`.
- **Core Functions**:
  - `cn(...inputs: ClassValue[])`: A utility function used across almost every component to handle dynamic Tailwind class merging safely.
- **State Management**: Local component state is managed using standard React Hooks (`useState`, `useEffect`). No global state management library (like Redux or Zustand) is used, as it is not necessary for this static structure.

## 5. Deployment & Build
To build the application for production:
```bash
npm run build
```
This runs the TypeScript compiler (`tsc`) and Vite's build process, outputting static, optimized HTML, JS, and CSS files into the `dist/` directory, which can be deployed to any static hosting service (Vercel, Netlify, GitHub Pages).
